# Alpha i18n Validation & End-to-End Test Plan

**CONTEXT:** We have built the frontend i18n infrastructure and the locale-aware backend generation engine. Now, we need to wire up a language selector UI component and execute an end-to-end integration test to confirm the translation data flow works flawlessly across the entire stack[cite: 3].

Follow these steps sequentially.

---

## STEP 1: Implement the Language Toggle UI Component
We need a simple way to change languages on the frontend that automatically updates our global state and our local KV User Ledger database.

1. Open your main header or root layout component (e.g., `apps/web/src/components/Header.tsx` or `OnboardingContainer.tsx`)[cite: 1, 3].
2. Add a clean language toggle element (e.g., an `EN / ES` button or dropdown).
3. **Behavior Rules:**
   - Clicking the toggle must trigger `i18next.changeLanguage(newLocale)` to instantly swap the frontend UI text[cite: 3].
   - It must update the local React state variable `locale` inside the onboarding session wrapper[cite: 3].
   - **Critical State Sync:** It must fire a `PATCH /api/ledger/:userId` request to save the updated `locale` string into our local `MM_LEDGER_ALPHA` KV namespace so the preference survives page refreshes and reaches the backend orchestrator[cite: 2, 3].

## STEP 2: Execute an End-to-End Spanish Funnel Run
With the toggle in place, run a full integration test through your local server[cite: 5].

1. Open your browser to your local environment (`http://127.0.0.1:8787` or your local Vercel port).
2. Switch the language toggle to **ES** right at Step 1.
3. Complete all 6 steps of the onboarding funnel using mock metrics[cite: 5].
4. **Verification Points:**
